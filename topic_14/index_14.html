Адаптивний Веб та Семантика Тексту

1. Семантичні Теги для Тексту

1.1. Цитати, Абревіатури та Адреси

У статті про розробку Тім Бернерс-Лі, батько Всесвітньої павутини, зазначив: Головна мета вебу — це доступність для всіх. Це коротка цитата, оформлена тегом <q>.

"Принцип модульності та розділення вмісту, стилю і поведінки є фундаментальним для створення довговічних і легко підтримуваних веб-систем."

— З повагою, розробник

Наш відділ HTML та CSS працює над цим. (Тег <abbr> показує повну назву при наведенні).

Контактна інформація організації, оформлена тегом <address>:

Київ, Україна,
Вулиця Програмістська, 5,
Індекс: 01001
Електронна пошта: info@example.com

1.2. Спеціалізовані Теги та Двонаправлений Текст

Цей текст є звичайною неправильною ціною новою ціною (видалення та вставка).

Для демонстрації двонаправленого тексту, ми використовуємо тег <bdo> (Bidirectional Override). Це корисно для відображення тексту, написаного справа наліво: Цей текст буде прочитано навпаки.

1.3. Інтерактивність без JavaScript (<details> та <summary>)

Елементи <details> та <summary> дозволяють створювати інтерактивні блоки, що розкриваються/згортаються.

Деталі про Адаптивність (Клікніть тут)

Адаптивний дизайн використовує CSS Media Queries для перевірки характеристик пристрою (наприклад, ширини вікна) і застосовує різні набори правил стилів відповідно до цих характеристик.

  • dir="rtl" - Ключ до успіху: мобільний підхід (Mobile-First).
  • Flexbox та Grid є основними інструментами адаптивного макетування.
Навіщо потрібні семантичні теги?

Семантичні теги покращують **доступність** (Accessibility) та **пошукову оптимізацію** (SEO).
